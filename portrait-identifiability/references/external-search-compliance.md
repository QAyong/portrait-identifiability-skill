# 外部检索与合规边界

使用反向图片搜索、人脸候选结果或敏感人脸数据前，先阅读本文件。本 skill 面向中国用户，默认使用中文说明和审慎合规表达。

## 首选架构

在 Codex 中做虚拟人物撞脸排查时，优先使用“浏览器反向图片搜索 + 本地候选比对”：

1. 预处理虚拟脸图片：裁剪到清晰人脸，必要时保留发际线和部分上下文，避免无关背景。
2. 通过 Codex 浏览器扩展打开百度识图。
3. 使用网页 UI 上传图片，等待结果页加载。
4. 通过浏览器 `pageAssets` 导出页面实际加载过的相似图片，并记录本地文件名和来源 URL。
5. 使用本 skill 的本地比对工具比较输入图与候选图。
6. 输出中文风险等级、依据、限制和人工复核建议，不确认身份。

不要把“内部人脸向量库”描述为默认首版架构。授权素材库、避让名单库、投诉库、项目角色库可以作为后续产品能力，但必须有明确授权、用途边界和数据治理。

## 可接受外部来源

- 通过 Codex 浏览器自动化访问百度识图，并从渲染后的结果页导出的候选图片。
- 通过 Playwright 打开百度识图网页，从浏览器网络响应和 DOM 图片地址中导出的候选图片。
- 用户提供的候选截图、图片 URL、页面链接、项目避让名单或授权素材。

不要把百度识图或任何网页反向图片搜索当作稳定 API。它们会受到公共网页 UI、浏览器状态、地区、账号、索引变化和访问限制影响。沙箱网络受限时，优先使用浏览器观察到的资源导出，不要依赖 Node 直接下载。

## 数据处理规则

- 本 Skill 的百度识图、反向图片搜索、外部候选采集和相关测试用例默认包含外部上传动作。用户触发这些能力时，视为已授权上传任务所需图片；不要再将上传确认作为阻塞步骤。
- 对多张图片、批量百度识图、批量反向搜索或同一测试目录内的连续测试，直接按任务范围执行；不要对同一批次内每张图重复发确认话术。
- 执行时可以简短说明将使用外部检索链路并继续操作。若外部服务失败、验证码阻断、网络不可用或页面结构变化，再报告失败原因并给出替代方案。
- 除非用户明确要求且确有工作产物需要，不要长期保存用户上传图片。
- 不要建立未经授权的公众人物人脸库。
- 不要批量抓取、保存或嵌入公开人脸图片。
- 不要暴露搜索结果中的私人个人信息。
- 仅在审查可追溯性需要时保留来源链接、页面标题和短摘录。
- 浏览器自动化只保留任务相关的结果标题、图片 URL、来源 URL 和简短页面线索。
- 优先做临时分析，避免不必要的持久化存储。

## 禁止用途

遇到以下请求时，应拒绝或转向合规替代方案：

- 通过图片识别私人自然人身份；
- 查找某人的住址、电话、社交账号、工作单位、学校、家属等个人信息；
- 跟踪、骚扰、监控或人肉搜索；
- 对未成年人做人脸搜索或身份识别；
- 将人脸相似度用于招聘、信贷、保险、住房等高影响决策；
- 输出确定性法律结论，例如确认侵权、确认不侵权、确认同一人。

## 安全表达

推荐使用：

- “外部检索返回若干相似图片，需要人工复核”
- “当前可访问检索范围内未发现明显高相似候选”
- “该候选与输入图在若干外部特征上相近”
- “本结果不能替代司法鉴定、律师意见或法院判断”

避免使用：

- “该人就是某某”
- “确认同一人”
- “确认侵权”
- “未侵权”
- “可以放心商用”

## 人工复核触发条件

以下情况必须建议人工复核：

- 目标图将用于广告、短剧、直播、电商、数字人、游戏角色、品牌 IP 或其他商业传播。
- 检索结果反复指向同一公众人物、网红、演员、主播、模特、运动员或已有商业角色。
- 候选对象在客户避让名单中，或涉及既往投诉/授权争议。
- 去除妆容、姿势、发型、光线和风格影响后，相似性仍较强。
- 检索范围不完整、不稳定或结果来源不足。
- 图片高度风格化，但仍保留真人标志性外部特征。
