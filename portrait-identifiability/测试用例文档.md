# 肖像权可识别性检测 — 测试用例文档

> 版本：v1.0 | 更新日期：2026-06-24

本文档用于搜集、整理和执行「肖像权可识别性检测」Skill 的用户级测试用例。所有用例从用户实际操作视角设计，覆盖主要功能链路和边界场景。

---

## 一、测试环境

| 项目 | 要求 |
|------|------|
| Python | 3.11+ |
| 依赖 | `pip install -r portrait-identifiability/requirements.txt` |
| 多模态引擎 | 豆包 (Doubao)，需设置环境变量 `DOUBAO_API_KEY` |
| 浏览器 | Chrome / Edge（百度识图链路需要 Playwright） |
| 操作系统 | Windows / macOS / Linux 均可 |

**环境准备命令：**

```bash
pip install -r portrait-identifiability/requirements.txt
playwright install chromium
```

**豆包 API 密钥：**

```bash
set DOUBAO_API_KEY=your-api-key-here
```

## 二、测试数据需求清单

在开始测试前，请收集以下类型的图片素材。每类至少准备 2-3 张，覆盖正常和边界情况。

### 2.1 真人照片

| 编号 | 类型 | 用途 | 要求 | 已收集？ |
|------|------|------|------|----------|
| R-01 | 正面清晰单人照 | 真人 vs 真人比对 | 正脸、五官清晰、光照均匀、无遮挡 | ☐ |
| R-02 | 正面清晰单人照（不同人） | 真人 vs 真人比对（低风险对照） | 与 R-01 明显不同的真人 | ☐ |
| R-03 | 与 R-01 同一人的不同照片 | 真人 vs 真人比对（高风险对照） | 同一人的另一张照片（不同角度/光线下） | ☐ |
| R-04 | 侧脸照片 | 边界测试 | 偏转角度 > 45° | ☐ |
| R-05 | 半遮挡照片 | 边界测试 | 口罩、墨镜、手部遮挡部分面部 | ☐ |

### 2.2 AI 生成虚拟人脸

| 编号 | 类型 | 用途 | 要求 | 已收集？ |
|------|------|------|------|----------|
| V-01 | AI 生成写实虚拟人 | 虚拟人撞脸排查 | 接近照片质感的 AI 生成人脸（如 Midjourney/SD 生成） | ☐ |
| V-02 | AI 生成写实虚拟人（另一张） | 批量排查 | 与 V-01 不同风格的虚拟人 | ☐ |
| V-03 | AI 生成低质量/失真虚拟人 | 边界测试 | 有明显生成瑕疵（眼睛不对称、皮肤塑料感等） | ☐ |
| V-04 | AI 生成多人物图 | 边界测试 | 一张图中有 2+ 张人脸 | ☐ |

### 2.3 风格化形象

| 编号 | 类型 | 用途 | 要求 | 已收集？ |
|------|------|------|------|----------|
| S-01 | 二次元/动漫风格角色 | 风格化 vs 真人比对 | 明显非写实风格（动漫、游戏角色） | ☐ |
| S-02 | 3D 渲染卡通角色 | 风格化 vs 真人比对 | 3D 卡通渲染（如 Pixar 风格） | ☐ |
| S-03 | 强风格化但保留真人特征 | 风格化 vs 真人比对（边界） | 滤镜/艺术风格化处理，但仍可辨认面部特征 | ☐ |

### 2.4 百度识图候选图片

> 这些图片将通过百度识图检索获得，无需提前准备。但需要准备一张用于触发检索的查询图。

| 编号 | 类型 | 用途 | 要求 | 已收集？ |
|------|------|------|------|----------|
| B-01 | 百度识图查询用虚拟人脸 | 触发百度识图采集 | 用于上传到百度识图的虚拟人脸图 | ☐ |

### 2.5 批量排查用图片集

| A-01 | 3-5 张虚拟人脸图片 | 批量排查 | 放在同一目录下，用于批量排查 | ☐ |

### 3.2 场景二：虚拟人脸 + 百度识图候选排查
